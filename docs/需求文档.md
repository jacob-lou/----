# 热点追踪系统 - 需求文档

## 1. 项目概述

热点追踪系统是一个实时热点信息聚合平台，通过多信息源采集全球热门话题，聚合分析后实时推送给用户。系统从多个平台（Google、Reddit、HackerNews、DuckDuckGo、Twitter/X）同步采集热点，每 30 分钟自动刷新一次。

## 2. 功能需求

### 2.1 多信息源热点采集

| 信息源 | 采集方式 | 数据类型 |
|--------|----------|----------|
| Google Trends | 网页爬虫 (无 API) | 热搜关键词 + 热度值 |
| Reddit | 网页爬虫 (Reddit JSON API) | 热门帖子 (标题/分数/评论数) |
| HackerNews | 公开 API (免费) | 热门故事 (标题/分数/评论数) |
| DuckDuckGo | 网页爬虫 (无 API) | 热搜关键词 |
| Twitter/X | twitterapi.io REST API | 热门趋势 + 热门推文 |

### 2.2 数据存储

- 热点记录持久化存储（SQLite）
- 去重机制：相同关键词/URL 不重复入库
- 记录每次采集的时间戳
- 支持历史热点回溯查询

### 2.3 实时推送

- 新热点到达时，通过 WebSocket 实时推送至前端
- 支持按信息源筛选
- 支持分页加载历史记录

### 2.4 定时任务

- 每 30 分钟自动触发一次全源采集
- 支持手动触发立即采集
- 采集失败时记录错误日志，不影响其他信息源

## 3. 非功能需求

- **频率控制**：网页爬虫控制请求频率，避免被封禁
- **容错性**：单一信息源失败不影响整体运行
- **可扩展性**：方便新增信息源
- **数据安全**：API Key 通过环境变量管理，不硬编码

## 4. 技术栈

| 模块 | 技术选型 |
|------|----------|
| 后端框架 | Express 5.x |
| ORM/数据库 | Prisma ORM + SQLite |
| 实时通信 | Socket.IO 4.x |
| 定时任务 | node-cron (cron 库) |
| 网页爬虫 | cheerio + node 原生 fetch |
| Twitter API | twitterapi.io (x-api-key 认证) |
| 前端 | 原生 HTML/CSS/JS + Socket.IO Client |
