## 3. 数据库设计 (Prisma Schema)

```prisma
datasource db {
  provider = "sqlite"
  url      = "file:./dev.db"
}

generator client {
  provider = "prisma-client-js"
}

model TrendItem {
  id          Int      @id @default(autoincrement())
  title       String                    // 热点标题
  url         String?                   // 原始链接
  source      String                    // 来源: google/reddit/hackernews/duckduckgo/twitter
  score       Int      @default(0)      // 热度分数
  extra       String?                   // JSON 额外数据
  externalId  String?                   // 外部唯一标识
  fetchedAt   DateTime @default(now())  // 采集时间
  createdAt   DateTime @default(now())  // 创建时间

  @@unique([source, externalId])        // 同源去重
  @@index([source])
  @@index([fetchedAt])
}

model FetchLog {
  id        Int      @id @default(autoincrement())
  source    String                      // 来源
  status    String                      // success / error
  message   String?                     // 错误信息
  count     Int      @default(0)        // 采集数量
  duration  Int      @default(0)        // 耗时 ms
  createdAt DateTime @default(now())
}
```

## 4. 信息源采集策略

### 4.1 Google Trends
- **方式**: 爬取 `https://trends.google.com/trending/rss?geo=US` (RSS Feed)
- **频率**: 每 30 分钟
- **解析**: 用 cheerio 解析 XML/RSS
- **去重**: 以 title 作为 externalId

### 4.2 Reddit
- **方式**: 访问 `https://www.reddit.com/r/popular/hot.json?limit=25`
- **频率**: 每 30 分钟
- **数据**: title, score, num_comments, permalink
- **去重**: 以 Reddit post id 作为 externalId

### 4.3 HackerNews
- **方式**: 官方 API `https://hacker-news.firebaseio.com/v0/topstories.json`
- **频率**: 每 30 分钟
- **数据**: 获取 top 30 story IDs，再批量获取详情
- **去重**: 以 HN story id 作为 externalId

### 4.4 DuckDuckGo
- **方式**: 爬取 `https://duckduckgo.com/` 首页或使用 `https://api.duckduckgo.com/?q=trending&format=json`
- **备选**: 抓取 DuckDuckGo 的自动补全接口 `https://duckduckgo.com/ac/?q=trending+topics&type=list`
- **去重**: 以关键词 hash 作为 externalId

### 4.5 Twitter/X (twitterapi.io)
- **方式**: REST API
- **认证**: `x-api-key` Header
- **端点**:
  - `GET https://api.twitterapi.io/twitter/trends?woeid=1` — 全球趋势
  - `GET https://api.twitterapi.io/twitter/tweet/advanced_search?query=...&queryType=Top` — 热门推文搜索
- **去重**: 以 trend name 或 tweet id 作为 externalId

## 5. API 设计

### REST Endpoints

| Method | Path | 描述 |
|--------|------|------|
| GET | `/api/trends` | 获取热点列表 (分页、筛选来源) |
| GET | `/api/trends/sources` | 获取所有可用来源 |
| POST | `/api/trends/refresh` | 手动触发采集 |
| GET | `/api/trends/stats` | 采集统计信息 |

### WebSocket Events

| Event | Direction | 描述 |
|-------|-----------|------|
| `new-trends` | Server → Client | 新热点推送 |
| `fetch-status` | Server → Client | 采集状态更新 |

## 6. 请求频率控制

- Google Trends RSS: 无严格限制，30 分钟一次足够
- Reddit JSON: 自带 User-Agent，30 分钟一次
- HackerNews: 公开 API 无限制
- DuckDuckGo: 轻量请求，30 分钟一次
- twitterapi.io: 1000+ req/sec，无需担心

## 7. 错误处理

- 每个信息源独立 try-catch
- 失败记录到 FetchLog 表
- 单源失败不影响其他源
- 前端通过 WebSocket 展示采集状态
